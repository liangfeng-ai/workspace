## 技术与论文

### 多 Agent 编排正在进入主流 API

OpenAI 在 GPT-5.6 API 中加入 Responses API 的多 Agent 编排 beta，说明“一个模型完成所有事”的开发方式正在变化。更现实的形态是：规划 Agent、执行 Agent、审查 Agent、检索 Agent 分工协作，并由平台提供状态、成本、工具调用和上下文管理。

核心价值：开发者可以构建更复杂的自动化系统，但也必须做评测、日志、权限和失败回滚，否则 Agent 会难以维护。

### Prompt Injection 检测成为 AI 安全基础设施

GitHub CodeQL 加入 AI prompt injection detection，意味着 LLM 应用安全会逐渐进入常规 DevSecOps。以后开发 AI 工具时，输入边界、工具权限、系统提示泄露、外部网页注入，都需要像 SQL 注入一样被系统性测试。

核心价值：AI 应用不再只是 prompt engineering，安全工程会成为上线门槛。

### Agent 成本控制成为平台级问题

OpenAI 的 prompt caching 和 reasoning 控制、GitHub 的 AI credit 管理、Qwen Cloud 的 Token Plan 都指向同一个方向：Agent 应用的成本必须可预测。未来优秀 AI 产品会把“智能程度”和“单位任务成本”一起优化。

核心价值：创业者不能只看模型能力，要设计任务路由、缓存、预算阈值和降级策略。

## 对我的启发

1. **开发者：** 尽快把 AI 编程工具从“聊天补全”升级为“可审查的 Agent 工作流”。例如让 Agent 开 issue、改代码、跑测试、生成 PR，但保留人类 review 和自动化测试闸门。

2. **开发者：** 新项目应默认加入 AI 安全检查：prompt injection、工具权限最小化、敏感信息过滤、日志审计。CodeQL 这类工具会变成标配。

3. **个人效率：** 选择模型时不要只追最强模型。日常任务可用便宜模型，高价值任务再切 Sol/Sonnet 这类高能力模型。建立自己的“模型路由习惯”能显著省钱。

4. **创业机会：** 企业 Agent 最大痛点不是 demo，而是可控上线：预算、权限、审计、失败恢复、可观测性。围绕这些做工具，比单纯封装聊天机器人更有机会。

5. **创业机会：** Microsoft 365、GitHub、Meta、Qwen 都在把模型嵌进现有工作流。垂直 SaaS 的机会在于：围绕一个行业数据和流程，做深度集成，而不是做通用助手。
