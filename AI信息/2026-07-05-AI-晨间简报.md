## 技术与论文

### Leanstral 1.5：AI 不只是“写答案”，而是让机器验证答案
Mistral 的 Leanstral 1.5 代表一个重要方向：让 AI 生成可由 Lean 4 检查的证明，而不是只输出看起来合理的解释。  
普通大模型写代码或数学证明时，可能“语气很确定但逻辑有漏洞”；形式化证明系统要求每一步都能被机器验证。因此，这类模型未来可能用于：
- 验证关键代码逻辑是否正确；
- 检查智能合约和金融系统的不变量；
- 把自然语言规格说明转为可执行验证目标；
- 降低形式化方法的使用门槛。

### OpenAI Codex 研究仍值得补读：Agent 使用正在从工程师扩散到全公司
- **发布时间**：2026-06-25。
- **核心结论**：OpenAI 的 Codex 使用数据显示，Agentic AI 正从编程工具变成通用工作工具。OpenAI 内部法律、招聘、财务等非工程岗位也在大规模使用 Codex；高频用户每天会并行运行大量长任务 Agent。
- **为什么今天仍重要**：Cursor iOS、Copilot Agent、企业会议 AI 控制等新闻都指向同一趋势：AI 的主战场正在从“问答”转向“委派任务”。
- **来源**：[OpenAI 官方文章](https://openai.com/index/how-agents-are-transforming-work/)、[arXiv 论文](https://arxiv.org/abs/2606.26959)

## 对我的启发

1. **开发者**：把工作流改成“任务队列 + Agent 并行执行”。例如：一个 Agent 改代码，一个 Agent 写测试，一个 Agent 查日志，一个 Agent 写变更说明。
2. **开发者**：生产系统不要绑定单一模型。Claude Fable 5 的恢复、Google 端点迁移、GLM-5.2 的低价竞争都说明：模型可用性、价格和合规状态会快速变化。
3. **个人效率**：会议 AI、移动端编程 Agent、Office Copilot 的价值不在“聊天”，而在减少切换成本。可优先尝试：会议纪要、邮件处理、代码 review、文档摘要。
4. **创业机会**：企业真正需要的是“可控 Agent”：有权限边界、预算上限、日志审计、模型路由和人工审批，而不是又一个聊天机器人。
5. **创业机会**：形式化验证 + AI 可能成为高端垂直市场，尤其适合智能合约、安全关键代码、金融规则引擎、医疗 / 航空等强合规领域。
