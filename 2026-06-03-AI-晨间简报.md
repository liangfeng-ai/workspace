# 今日 AI 简报：2026-06-03

## 一句话总结

今天的主线是：**AI 正从“聊天 / 编程助手”快速升级为企业可治理的 Agent 平台**。Microsoft Build 2026、OpenAI Codex 更新、Perplexity Search as Code 都指向同一趋势：模型本身仍重要，但更大的竞争焦点正在变成 **上下文、工具调用、企业权限、部署环境和可控执行**。

### 6. Perplexity 提出 Search as Code：让 Agent 直接生成搜索流水线代码

- **发布时间**：2026-06-01；6 月 2 日被开发者社区广泛讨论
- **摘要**：Perplexity Research 发布 **Search as Code** 架构：Agent 不再多轮调用搜索工具，而是生成 Python 搜索流水线，在沙箱中执行检索、解析、筛选和聚合。
- **为什么重要**：这可能是 Agent 工具调用的一条重要路线：从“模型一步步调用工具”变为“模型生成可执行检索程序”。对研究 Agent、自动化情报、合规监控、金融信息检索很有启发。
- **来源**：[Rethinking Search as Code Generation](https://research.perplexity.ai/articles/rethinking-search-as-code-generation)

## 技术与论文

### Search as Code：Agent 检索的新范式

Perplexity 的核心想法是：复杂搜索任务不一定要靠模型一轮轮调用工具，而可以让模型先生成一个搜索程序，再在沙箱中运行。这样做的优势是：

- 搜索步骤更清晰，可复现
- 可以并行抓取、筛选、排序
- 适合复杂研究任务
- 更容易调试和优化

对开发者来说，这提示我们：**Agent 的下一步不是更多工具，而是让 Agent 写出可执行工作流**。

### MiniMax Sparse Attention：为 1M 上下文降低成本

MiniMax M3 使用其 MSA 架构，官方称在 1M 上下文长度下，每 token 计算量约为上一代模型的 1/20。普通用户可以理解为：模型能读更长内容，同时不至于让推理成本爆炸。

如果第三方验证成立，这对长上下文应用很关键，例如：

- 读完整代码库
- 读数百页合同
- 长视频理解
- 多轮 Agent 任务记忆

### Agentic AI 用于真实 A/B 实验设计

- **论文发布时间**：2026-06-01
- **论文**：Beyond One-shot: AI Agents for Learning in Field Experiments
- **核心价值**：研究者在医疗处方消息场景中测试了 AI Agent 是否能从上一轮实验数据中学习，并自动生成下一轮更好的干预文案。结果显示，带工具的 Agent 能基于真实实验数据生成更有效方案。
- **非专业解释**：AI 不只是帮你“想几个文案”，而是能看历史实验数据，总结哪些策略有效，再设计下一轮实验。
- **来源**：[arXiv: Beyond One-shot](https://arxiv.org/abs/2606.02458)

## 对我的启发

1. **开发者：优先学习“Agent 可控执行”而不是只学提示词**  
   重点关注沙箱、权限、工具调用、审计日志、工作流生成、回滚机制。Microsoft Agent 365、Perplexity Search as Code 都说明这会成为核心能力。

2. **个人效率：把 Codex 当成“内部工具生成器”使用**  
   不要只让它写代码。可以让它生成周报模板、数据仪表盘、调研页面、自动化脚本、客户资料整理工具。

3. **创业机会：企业内部轻应用会被 AI 重新定义**  
   ChatGPT Sites、Rayfin、Replit 集成、Copilot App 都在抢“从想法到内部应用”的入口。机会在于：行业模板、权限治理、数据连接器、审计合规、垂直场景工作流。

4. **产品策略：模型差异会缩小，数据和上下文差异会放大**  
   Microsoft IQ / Work IQ 的方向很明确：谁掌握企业上下文，谁就能做更有用的 Agent。

5. **教育 / 青少年 AI 产品要提前做安全设计**  
   OpenAI 的青少年安全倡议说明监管会更细。做学习助手、儿童陪伴、家教 Agent 的团队，应尽早加入年龄分级、家长控制、数据最小化和安全评测。
